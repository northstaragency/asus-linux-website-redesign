Required for proprietary NVIDIA drivers.

Run this command:
sudo dnf install https://mirrors.rpmfusion.org/free/fedora/rpmfusion-free-release-$(rpm -E %fedora).noarch.rpm https://mirrors.rpmfusion.org/nonfree/fedora/rpmfusion-nonfree-release-$(rpm -E %fedora).noarch.rpm

Enable the COPR repository

Add the asus-linux repository to access ROG packages.

sudo dnf copr enable lukenukem/asus-linux

Repository enabled successfully.

Install ROG packages

Install asusctl and supergfxctl for hardware control.

sudo dnf install asusctl supergfxctl

Enable services

Start the background daemons for GPU switching and power profiles.

Frequently Asked Questions

+ What does ROG Control Center do?
ROG Control Center gives you full control over your ASUS ROG laptop's hardware on Linux. Switch between integrated and dedicated GPUs, customize fan curves, manage battery charge limits, and create performance profiles‚Äîall without touching config files or terminal commands.
+ Does it work with my model?
Most ASUS ROG laptops from 2018 onwards are supported, including G14, G15, Flow X13, Zephyrus, and Strix series. Check the Models page for your specific laptop and see which features are available. If your model isn't listed, you can report it to help expand support.
+ Is it safe? Can I uninstall it?
Yes. All changes are reversible, and the app includes confirmations for risky actions. To uninstall, simply remove the packages using your package manager: sudo pacman -R asusctl supergfxctl (Arch) or equivalent for your distro. Your system returns to defaults.
+ What if I need help or encounter an error?
Check the Playbooks section for step-by-step troubleshooting guides. For detailed help, visit asus-linux.org/guides. You can also join the community Discord or GitHub discussions for real-time support.
+ Will this break Secure Boot or dual-boot with Windows?
ROG Control Center works with Secure Boot enabled on most distros. For dual-boot setups, GPU switching may require a reboot, but Windows will recognize the changes. If you encounter issues, check the Playbooks for specific guides on Secure Boot configuration and dual-boot GPU management.
